5. COMMUNITY PULSE & RISKS (The "Don't Do It" List)

Estado Categoría Detalle del Pulso / Riesgo
🔴 Bugs Trending Problemas de rendimiento en tableros Kanban/Scrum con muchos filtros rápidos (Jira Cloud): Varios usuarios reportan ralentizaciones significativas al cargar tableros que utilizan más de 5 filtros rápidos complejos, especialmente con la nueva interfaz de "Quick Filters 2.0". Esto impacta la productividad diaria de los equipos de desarrollo.
Insight Consultor: ACCIÓN CRÍTICA: Aconsejar a los clientes evitar la sobrecarga de filtros rápidos complejos en tableros de alta visibilidad. Evaluar la necesidad real de cada filtro y considerar consolidar o usar filtros de búsqueda globales. Investigar workarounds y monitorear el progreso del fix por parte de Atlassian. Riesgo: Deterioro de la experiencia de usuario y frustración del equipo.
🟡 Deprecaciones Deprecación de la API REST jira/rest/api/2/user para búsqueda de usuarios por email sin autenticación (Cloud): A partir del 1 de octubre de 2026, esta API requerirá siempre autenticación (OAuth/API Token) incluso para resultados limitados.
Insight Consultor: ALERTA ARQUITECTÓNICA: Esto es un breaking change para integraciones heredadas o apps custom que pudieran estar haciendo llamadas anónimas a esta API para resolver usuarios. Se debe auditar cualquier integración que busque usuarios por email. Es una oportunidad para revisar y modernizar la seguridad de las integraciones, migrando a autenticación adecuada (OAuth 2.0 o API Tokens) antes de la fecha límite para evitar interrupciones.
🟡 Discusiones Debate sobre la dificultad de migrar desde Jira Server/Data Center a Cloud con esquemas de proyectos muy complejos: La comunidad sigue expresando frustración por las limitaciones del JCMA para migrar configuraciones de proyectos con muchos workflows compartidos, campos personalizados globales y resoluciones "in-flight".
Insight Consultor: OPORTUNIDAD/RIESGO: Esta discusión subraya la necesidad de una consultoría experta en migraciones Cloud. Vende la "evaluación de complejidad de migración" como servicio inicial para identificar estos desafíos temprano. Presenta la posibilidad de "limpiar" el entorno Data Center antes de migrar (refactorización de esquemas, consolidación) como un servicio de valor añadido. Riesgo: Si no se gestiona proactivamente, puede llevar a migraciones fallidas o con alto coste de retrabajo post-migración.
