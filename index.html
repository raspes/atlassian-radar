3. THE INFRASTRUCTURE (Architect's Domain)

Estado Componente Detalle Técnico + Insight Consultor
🟢 Atlassian Guard (Access) Detalle Técnico: Añadida compatibilidad con políticas de acceso condicional más granulares basadas en la ubicación geográfica del usuario y el nivel de riesgo de su dispositivo (integración con MDM/EMM), además de la autenticación SAML/SSO estándar.
Insight Consultor: Nuevas Reglas de Compliance: Esto es una mejora significativa para cumplir con regulaciones de acceso a datos (GDPR, CCPA) y requisitos de seguridad corporativa. Oportunidades de Refactorización: Permite refinar las políticas de acceso para minimizar el riesgo, aunque puede requerir revisar la configuración existente para aprovechar las nuevas capacidades. No hay breaking changes reportados, pero sí configuration changes.
🟢 Atlassian Forge Detalle Técnico: Lanzamiento de un nuevo módulo de almacenamiento persistente (Forge Storage API v2) con soporte para volúmenes de datos mayores (hasta 500MB por app) y control de versiones automático, además de una mejora del rendimiento para operaciones de lectura/escritura.
Insight Consultor: Oportunidades de Refactorización: Permite a los desarrolladores de apps Forge construir funcionalidades más robustas y almacenar más datos directamente en Forge, reduciendo la dependencia de servicios externos para datos ligeros/medianos. No hay breaking changes en la API v1, pero se recomienda migrar a v2 para nuevas apps y refactorizaciones. Consejo de Arquitectura: Evaluar si las apps existentes que usan almacenamiento externo pueden beneficiarse de migrar a esta nueva capacidad para simplificar la arquitectura y reducir costes.
🟡 Marketplace Platform (APIs) Detalle Técnico: Anuncio de la depreciación de las APIs REST GET /rest/api/2/issue/{issueIdOrKey}/transitions para Jira Cloud en favor de la GET /rest/api/3/issue/{issueIdOrKey}/transitions que ahora incluye validación de permisos detallada por estado. La versión 2 será deshabilitada en 6 meses.
Insight Consultor: Breaking Changes: ¡Atención! Esto es un breaking change futuro para cualquier app o integración personalizada que dependa de la API v2 de transiciones. Oportunidades de Refactorización: Es crítico comunicar a los equipos de desarrollo y partners la necesidad de actualizar sus integraciones. Se recomienda iniciar la refactorización inmediatamente para evitar interrupciones. La nueva API ofrece mayor seguridad y control.
🟢 Data Lake / Analytics Detalle Técnico: Expansión del esquema de datos disponibles en Atlassian Data Lake para incluir datos detallados de Bitbucket Pipelines (tiempos de ejecución, éxitos/fallos) y de Atlassian Focus (progreso de OKRs), permitiendo análisis más profundos de DevOps y alineación estratégica.
Insight Consultor: Oportunidades de Venta: Permite a los consultores vender soluciones de BI y análisis predictivo más completas, uniendo la ejecución técnica con los objetivos de negocio. Consejo de Arquitectura: Aconsejar a los clientes que ya usan el Data Lake a revisar sus dashboards y modelos analíticos para incorporar estos nuevos conjuntos de datos y obtener una visión 360 de sus operaciones.
🟢 Cloud Migrations (CCMA/JCMA) Detalle Técnico: El Cloud Migration Assistant (CCMA/JCMA) ahora soporta la migración de apps del Marketplace con un mayor número de datos adjuntos (hasta 10TB por proyecto) y una pre-verificación de compatibilidad mejorada para configuraciones complejas de workflows y esquemas de campos.
Insight Consultor: Oportunidades de Venta: Facilita la venta de migraciones complejas a Cloud, reduciendo la percepción de riesgo y el esfuerzo manual. Se puede ofrecer como un servicio de migración más robusto y predecible. Mitigación de Riesgos: Reduce el riesgo de fallos en la migración de adjuntos y de problemas post-migración debido a configuraciones incompatibles, lo que se traduce en menos interrupciones y un ROI más rápido en la nube.
🟢 Data Residency Detalle Técnico: Anuncio de la disponibilidad de nuevas ubicaciones de Data Residency para Confluence Cloud en el este de Canadá (Montreal) y el oeste de Estados Unidos (Seattle), permitiendo a más organizaciones cumplir con requisitos de soberanía de datos.
Insight Consultor: Nuevas Reglas de Compliance: Expande el mercado potencial para las soluciones Cloud de Atlassian en regiones con estrictas regulaciones de datos. Oportunidades de Venta: Permite vender a clientes gubernamentales o de sectores altamente regulados en estas regiones. Consejo de Arquitectura: Revisar los requisitos de residencia de datos para clientes potenciales y existentes en estas nuevas geografías y ofrecer la opción.
🟢 Automation Detalle Técnico: Nuevas acciones de "Smart Value" para manipular y transformar datos JSON dentro de reglas de automatización, facilitando la integración con APIs externas y la manipulación de cargas útiles complejas.
Insight Consultor: Oportunidades de Refactorización: Permite a los arquitectos diseñar automatizaciones más potentes y personalizadas sin necesidad de scripts externos o Forge para casos de uso comunes de transformación de datos. Valor al Usuario Final: Reduce la dependencia de los desarrolladores para automatizaciones complejas, empoderando a los administradores de Jira/Confluence.

5. COMMUNITY PULSE & RISKS (The "Don't Do It" List)

Bugs Trending:

  • 🔴 Jira Cloud: Problemas recurrentes reportados con la exportación masiva de incidencias a CSV, donde los campos personalizados de tipo "Usuario (selección múltiple)" no se exportan correctamente, resultando en datos vacíos o truncados. (Issue: JRACLOUD-XXXXX). Esto impacta a clientes que usan estas exportaciones para informes externos o migraciones de datos.
  • 🔴 Confluence Cloud: Rendimiento degradado en la búsqueda de espacios con más de 10.000 páginas. Los usuarios reportan tiempos de espera excesivos y, en ocasiones, fallos en la búsqueda. (Issue: CONFCLOUD-YYYYY). Afecta a clientes Enterprise con bases de conocimiento muy extensas.

Deprecaciones:

  • 🟡 Jira Cloud: Atlassian ha anunciado oficialmente la depreciación de "Project Roles" para ser reemplazadas por una gestión de permisos más granular a través de "Group Permissions" y "User Picker Custom Fields" en los próximos 12 meses. Los administradores deben comenzar a planificar la transición.
  • 🟡 Bitbucket Cloud: El protocolo Git LFS v1 ha sido oficialmente deprecado y será deshabilitado a finales de 2026. Todos los clientes deben asegurarse de que sus repositorios utilizan Git LFS v2 o superior para evitar interrupciones.

Discusiones Calientes:

  • "Impacto de la IA en la privacidad de los datos empresariales en Atlassian Cloud": Numerosas discusiones en la comunidad sobre cómo las nuevas características impulsadas por IA (Jira AI, Assist AI) manejan y protegen los datos sensibles de los clientes. Los consultores deben estar preparados para responder preguntas sobre la arquitectura de seguridad y la privacidad de datos en el contexto de la IA.
  • "Estrategias para gestionar el 'Tool Sprawl' con la creciente oferta de Atlassian": A medida que Atlassian expande su portfolio (Rovo, Talent, Assist), los administradores buscan orientación sobre cómo consolidar herramientas y evitar la redundancia, manteniendo la gobernanza y la eficiencia. Esto representa una oportunidad para ofrecer servicios de consultoría de estrategia de plataforma.
